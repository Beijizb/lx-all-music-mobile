# 现代化 UI 诊断指南

## 问题：打开软件没有看到变化

### 可能的原因

#### 1. 使用的是旧版本 APK ⚠️

**症状**:
- 播放器界面没有变化
- 列表项样式没有变化
- 没有看到新的动画效果

**解决方案**:
1. 等待 GitHub Actions 构建完成（通常需要 3-5 分钟）
2. 在 GitHub 仓库的 Actions 页面查看构建状态
3. 构建成功后，下载新的 APK
4. 卸载旧版本，安装新版本

**如何确认版本**:
- 打开应用设置
- 查看版本号是否为 `2.0.0.8` (versionCode: 79)
- 如果版本号较低，说明使用的是旧版本

---

#### 2. 组件未正确加载 🔍

**检查方法**:

1. **检查播放器详情页**
   - 打开任意歌曲
   - 查看封面是否有旋转动画
   - 查看进度条是否更粗（8px Material / 6px Cupertino）
   - 查看控制按钮是否更大（60-80px）

2. **检查音乐列表**
   - 进入"我的音乐"
   - 查看列表项是否有音质徽章（SQ/HQ/VIP）
   - 点击列表项是否有按压动画
   - 查看是否有更好的视觉层级

3. **检查主题切换**
   - 进入设置 → 基本设置 → 主题框架
   - 切换 Material Design / iOS Cupertino
   - 查看组件圆角和样式是否变化

---

#### 3. 缓存问题 🗑️

**解决方案**:
```bash
# 清除 Metro bundler 缓存
npm start -- --reset-cache

# 或者清除 Android 构建缓存
cd android
./gradlew clean
cd ..
```

---

#### 4. 构建失败 ❌

**检查方法**:
1. 访问 GitHub 仓库
2. 点击 "Actions" 标签
3. 查看最新的 workflow 运行状态
4. 如果显示红色 ❌，说明构建失败

**常见构建错误**:
- 模块导入路径错误（已修复）
- TypeScript 类型错误
- 依赖缺失

## 如何获取最新版本

### 方法 1: GitHub Actions（推荐）

1. 访问: https://github.com/Beijizb/lx-b/actions
2. 点击最新的 "Build Android APK" workflow
3. 等待构建完成（绿色 ✅）
4. 下载 Artifacts 中的 APK
5. 安装到手机

### 方法 2: 本地构建

```bash
# 1. 拉取最新代码
git pull origin main

# 2. 安装依赖
npm install

# 3. 清除缓存
npm start -- --reset-cache

# 4. 构建 APK
cd android
./gradlew assembleRelease
cd ..

# 5. APK 位置
# android/app/build/outputs/apk/release/app-release.apk
```

## 常见问题

### Q: 为什么我看不到任何变化？

**A**: 最可能的原因是您使用的是旧版本 APK。请确认：
1. 版本号是否为 `2.0.0.8` 或更高
2. GitHub Actions 构建是否已完成
3. 是否下载并安装了最新的 APK

### Q: 如何确认新组件已加载？

**A**: 最简单的方法：
1. 打开任意歌曲
2. 查看封面是否旋转（播放时）
3. 如果旋转，说明新组件已加载

### Q: 主题切换后没有变化？

**A**:
1. 确认版本号是否正确
2. 尝试重启应用
3. 检查设置中的主题框架选项

### Q: 列表滚动不流畅？

**A**:
1. 确认使用的是最新版本
2. FlatList 优化已在 v2.0.0.7 中应用
3. 如果仍然卡顿，可能需要集成图片缓存（待完成）
